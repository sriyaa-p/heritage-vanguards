From Pipeline to Protocol
MCP
Server

With the core multi-agent pipeline complete, we needed a way for external AI agents to interact with Heritage Sentinel programmatically — without coupling them to our internal REST API.

We built backend/mcp_server.py — a full Model Context Protocol server that exposes our entire pipeline as 6 callable tools. Any MCP-compatible AI agent (Gemini via ADK, Claude Desktop, or any future agent) can now list submissions, retrieve dossiers, search the UNESCO registry, fetch scoring criteria, submit new candidate sites, and record human review decisions — all through a single standardised interface.

The server supports two transport modes: stdio for local agent use and Claude Desktop integration, and sse for remote agents connecting over HTTP on port 8001.
